3 Implementation Approaches

Choose how ambitious V0 should be

A

Minimal Wire-Up — 2–3 days

Scope: Add MP4/WebM export only to the existing Export modal. No GIF (quality risk). No animated preview.

How it works: User clicks "Export as MP4" → progress bar → file downloads. They don't see the animation before export.

New code: ~160 lines total. One new file (storyAnimationExport.ts) + ExportModal changes.

Pros

  • Fastest to validate
  • Reuses 100% of existing pipeline
  • Zero new UI concepts

Cons

  • Users export blind (no preview)
  • No GIF = no Twitter/Slack inline
B

MP4 + Animated Preview — 5–7 days Recommended

Scope: MP4 export + an animated canvas preview inside the export modal so users see the animation before downloading.

How it works: Export modal shows a small looping animation preview (200×200px canvas running the full story). Below it: "Download MP4" button.

New code: ~280 lines. Sequencer + wiring + preview canvas component.

Pros

  • Users see output before exporting
  • Higher confidence → more sharing
  • Still fast to build

Cons

  • 2× build time vs A
  • Preview canvas adds complexity
C

MP4 + GIF + Preview — 10–14 days

Scope: Full animated export with both formats, preview, and quality optimization for GIF (dithering, dimension capping, format-specific color profiles).

How it works: Export modal shows format tabs (MP4 / GIF), quality settings per format, animated preview, and download.

New code: ~500 lines. Significantly more complexity in GIF pipeline.

Pros

  • Full format coverage
  • GIF = Twitter/Slack/Discord inline

Cons

  • GIF banding on dark gradients
  • High risk of over-engineering
  • 2–3 week build for a hypothesis

💡 Recommendation: B if you want user confidence before sharing. A if you want pure speed-of-validation.
